01

WebGL Rendering Pipeline

WebGL issues GPU draw calls via JavaScript. Each frame, Three.js traverses the scene graph, batches geometries, uploads uniform data (matrices, lights), then executes vertex shaders to transform 3D coordinates into clip-space, followed by rasterization and fragment shaders that compute per-pixel color. Draw calls spike when multiple meshes with unique materials are visible simultaneously.

02

How Three.js Abstracts WebGL

Without Three.js, each mesh requires manual VAO/VBO setup, shader compilation, and matrix math. Three.js compiles MeshStandardMaterial into a complete GLSL program internally, manages the WebGLRenderer state machine, and handles the requestAnimationFrame loop. The triangle count reflects vertex buffer sizes Three.js maintains on the GPU.

03

Active Materials

MeshStandardMaterial — PBR: metalness + roughness uniforms fed to the BRDF shader.
MeshPhongMaterial — Phong model: ambient + diffuse + specular terms, cheaper than PBR.
MeshLambertMaterial — diffuse-only, computed per vertex. Minimal fragment shader cost.
Wireframe — GL_LINES draw mode, bypasses fragment shading entirely.
